# dir-watch — утилита контроля изменений в каталоге

## Описание

`dir-watch` — консольная утилита для периодической фиксации состояния каталога
и выявления изменений в нём: новых, изменённых и удалённых файлов.

Особенности:
- работает полностью локально, сеть не требуется;
- написана на C++ (`<filesystem>`, `<fstream>`, `<unordered_map>`, `<getopt.h>`);
- бинарник скомпилирован под Debian 11, совместим с Debian 11, Debian 12,
  Astra Linux 4.7 и Astra Linux 4.8.

## Установка

При наличии прав `sudo` рекомендуется устанавливать утилиту через
прилагаемый скрипт:

```bash
chmod +x install.sh
sudo ./install.sh
```

## Использование

```
dir-watch [options]

Options:
  -s, --snapshot <directory>   Создать снимок состояния каталога
  -c, --compare <directory>    Сравнить каталог с последним снимком
  -e, --export <filename>      Выгрузить отчёт последнего сравнения в файл
  -h, --help                   Показать справку
```

### Примеры

**1. Создание начального снимка**

```bash
dir-watch -s /home/user/project
dir-watch --snapshot /home/user/project
```

Внутри указанного каталога создаётся скрытый файл `.snapshot`, где для
каждого файла хранится путь, размер и время последнего изменения.

**2. Повторное сканирование и сравнение**

```bash
dir-watch -c /home/user/project
dir-watch --compare /home/user/project
```

Программа сравнивает текущее состояние каталога с сохранённым снимком по
размеру и времени модификации, выводит список новых / изменённых /
удалённых файлов и сводку в консоль. Результат сравнения также сохраняется
во временный файл `/tmp/.dir-watch-changes`.

**3. Экспорт отчёта**

```bash
dir-watch -e report.txt
dir-watch --export report.txt
```

Копирует содержимое последнего отчёта о сравнении (`/tmp/.dir-watch-changes`)
в указанный файл — в примере это `./report.txt` в текущей рабочей директории.
Команду `--export` можно вызывать независимо, без повторного указания
каталога — она всегда берёт результат последнего выполненного `--compare`.

**4. Справка**

```bash
dir-watch -h
dir-watch --help
```

## Как это устроено

- **`scan_directory`** — рекурсивно обходит каталог (`recursive_directory_iterator`),
  собирает метаданные (путь, размер, время модификации) по каждому обычному
  файлу. Файлы,создаваемые утилитой пропускаются, чтобы не попадать в сравнение самим с собой.
- **`save_snapshot` / `load_snapshot`** — сохраняют и читают снимок в текстовом
  формате `путь|размер|время_изменения`, по одной записи на строку. При
  загрузке снимок раскладывается в `std::unordered_map` по пути файла —
  это даёт поиск и сравнение за O(1) на файл.
- **`compare_snapshots`** — проходит по текущим файлам: если файла не было
  в старом снимке — он новый; если был, но изменился размер или время —
  изменённый; после прохода все оставшиеся в хеш-таблице записи — это файлы,
  которые были в снимке, но пропали из каталога, то есть удалённые.
- **`export_report`** — построчно копирует последний отчёт сравнения в
  указанный пользователем файл.
